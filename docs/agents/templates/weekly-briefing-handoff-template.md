# Weekly Briefing Handoff Template

Use this when any agent hands a completed, blocked, repeated, or high-risk task to Weekly Briefing Agent.

```text
Weekly Briefing Handoff

Task reference:
[Task ID or AgentMail thread ID]

Agent:
[Agent name]

Area:
[Security / Revenue / Auth / Reports / Support / Quality / Admin]

Risk:
[P0 / P1 / P2 / P3]

Status:
[done / blocked / approval_needed / monitoring]

Summary:
[One or two sentences.]

Customer impact:
[none / single_customer / multiple_customers / unknown]

Revenue impact:
[none / possible / confirmed / unknown]

Decision needed:
[None or exact decision]

Recommended next priority:
[keep / escalate / close / monitor]

Include priority:
[critical / high / normal]
```

## Include Rules

| Include priority | Use when |
| --- | --- |
| critical | P0 incident, security risk, live payment issue, production outage, customer data issue. |
| high | P1 pattern, unresolved complaint, repeated report failure, checkout issue, deployment warning. |
| normal | Routine billing movement, support pattern, report-quality note, completed operational task. |

One-off P3 items do not need briefing unless they repeat.
